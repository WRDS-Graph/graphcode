### (2) The seam to split along

**Keep `CytoscapeGraphRenderer` as the facade. The natural seams, by decreasing safety, mapped to actual line ranges:

### (4) Top risk

**The 64 mutable private fields are a hidden shared-state graph; the comparison-overlay and expansion clusters read/write overlapping maps (`nodeMap`, `childrenMap`, `expandedNodes`, `overlayElementIds`, `expandedClusters`, the `comp*` mirrors).** Extracting a stateful cluster into a separate object risks **silently splitting that shared state into two divergent copies** — the facade mutates its `expandedNodes` while the extracted module mutates a stale one, producing graphs that look right on first render but desync on the *second* expand/overlay interaction. Unit tests that do one render won't catch it.

**Mitigations baked into the plan:** (a) Tier A (pure, no shared state) ships first and is most of the line-count win with near-zero risk; (b) the `RenderContext` object (step 4) makes shared state a single owned reference passed by identity, so extracted modules mutate *the same* maps, not copies; (c) the smoke test in step 7 must exercise **sequential** interactions (expand → overlay → expand again → reset), plus the Cytoscape zoom/pan preservation rule from `CLAUDE.md §6.4`, which `resolveOverlaps`/drag/layout can break.
